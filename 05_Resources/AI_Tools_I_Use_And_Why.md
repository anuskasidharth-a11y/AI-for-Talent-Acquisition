# AI Tools I Use and Why

## Introduction

I do not use AI tools because they are trendy. I use each one because it solves a specific problem in my recruiting workflow faster or more accurately than doing it manually. Below is my actual stack, what each tool is for, and where I have been burned by relying on it too much.

## My AI Recruiting Stack

### 1. ChatGPT
**Why I use it:** first drafts. Job descriptions, outreach sequences, interview scorecards, boolean strings I want a second opinion on.
**Best use case:** turning a messy intake call into a structured job description in minutes instead of an hour.
**Limitation:** it will confidently invent a certification, a tool name, or a market rate if I do not correct it. I never send anything it wrote to a candidate or client without reading every line myself.

### 2. Claude
**Why I use it:** longer, more nuanced writing where tone matters, like a rejection message to a finalist candidate or a difficult update to a hiring manager.
**Best use case:** when I need something written that has to sound human and empathetic, not templated.
**Limitation:** slower for quick one-line boolean tweaks compared to ChatGPT, so I do not use it for fast iterative sourcing work.

### 3. Perplexity
**Why I use it:** verifying claims. If a candidate says they hold a patent, led a specific launch, or worked at a company under an NDA-sensitive project name, I check it here before it goes into a submission.
**Best use case:** I once had a candidate claim a specific patent filing on their resume. A two-minute Perplexity check surfaced the actual filing under a slightly different name, which meant I could confirm it accurately instead of taking the resume at face value or rejecting a true claim over a naming mismatch.
**Limitation:** it can still miss recent or paywalled sources, so a clean result is not a guarantee, only a stronger signal.

### 4. Gemini
**Why I use it:** cross-checking an answer I got from ChatGPT or Claude when the stakes are high, like compensation benchmarking for a client conversation.
**Best use case:** getting a second model's take before I put a number in front of a hiring manager.
**Limitation:** I do not use it as a primary tool. It is a second opinion, not a first draft engine, in my workflow.

### 5. NotebookLM
**Why I use it:** digesting long client documents, like a 40-page role requirement doc or a full org chart briefing, before an intake call.
**Best use case:** walking into an intake call already knowing the client's structure instead of asking basic questions that waste their time.
**Limitation:** only as good as the documents I feed it. Garbage or outdated source docs produce a confident but wrong summary.

## Comparison Table

| Task | Tool I Reach For |
|---|---|
| Drafting a job description | ChatGPT |
| Writing a sensitive candidate message | Claude |
| Verifying a candidate's claim | Perplexity |
| Compensation benchmarking sanity check | Gemini |
| Digesting a long client document | NotebookLM |
| Boolean string iteration | ChatGPT |

## Lessons Learned

The biggest lesson has been that speed and verification are two different jobs. AI makes the first draft fast. It does not make the fact-checking fast, and skipping that step is where I have seen recruiters get burned, including myself early on. The time I save on drafting, I now spend on verification. Net time saved is real, but it is smaller than it looks if you are doing this responsibly.
